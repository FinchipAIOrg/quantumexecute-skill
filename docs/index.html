Skill Profile

What this skill does

Guides AI agents through crypto algorithmic execution workflows — account discovery, market data pre-checks, TWAP/VWAP/POV order creation, lifecycle management (update/cancel/fills), balance and position queries, and TCA analysis with Excel exports. Multi-exchange: Binance, OKX, LTP, Deribit, Hyperliquid.

When to use it

User needs an AI agent to execute large crypto orders with algorithmic precision — split orders intelligently, track execution quality, manage positions across exchanges, or generate TCA reports. Designed for quantitative teams and institutional-grade trading workflows.

What you get

Script-based execution with hard P0 rules: no fabricated values, confirmation gates on all mutating operations, numeric precision preserved from script output, credential isolation in env vars only, and fixed response templates for order completion and TCA summaries.

Who This Skill Is

QuantumExecute (QE) is built for quantitative trading teams and institutional traders who need agent-driven execution with full audit trail. Every order operation runs through Python scripts — the agent orchestrates, scripts execute. Script stdout is authoritative; never fabricate values.

QE connects through qe-connector (Python SDK). Exchange API keys are bound in the QE platform — the agent never touches raw exchange credentials.

Best For

Systematic funds, treasury operations, market makers, and quant desks running schedule-based or benchmark-sensitive execution. Large orders that need smart splitting to reduce market impact. Multi-exchange portfolio rebalancing with TCA verification.

Hard Rules (P0)

Safety — 7 Non-Negotiable Guardrails

Scripts execute first — no fabricated values.

Every numeric output, balance, price, fill quantity, and PnL must come from script stdout. Never invent or estimate values. Script output is the single source of truth.

Confirmation gates on all mutating operations.

Before any create, update, or cancel order operation, require explicit user confirmation. Apply trading risk checks and parameter validation from references/trading.md before order writes.

Preserve numeric precision.

Keep total, page, pageSize, and all pagination fields exactly as returned by scripts. No rounding, no truncation.

Report failures — don't hide them.

When scripts fail, report stderr and error codes to the user. Do not silently skip or retry without informing. Error handling per references/errors.md.

Credentials in env vars only.

QE_API_KEY and QE_API_SECRET must stay in environment variables. Never print, never inline in code, never expose in chat. Use restricted QE keys with exchange API IP whitelisting and minimal permissions.

Pre-checks before execution.

Always run list_exchange_apis.py for account discovery before choosing an account-specific script. Always run market-data and account pre-checks before any create/update/cancel operation.

Fixed response templates.

Use response templates from references/output-formats.md for order completion summaries and TCA reports. Consistent output format across all exchange and algorithm combinations.
